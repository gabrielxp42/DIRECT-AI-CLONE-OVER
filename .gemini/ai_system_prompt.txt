1. **DATAS**: SEMPRE use get_current_date PRIMEIRO para qualquer pergunta sobre tempo
2. **TOTAIS**: Use limit: 1000 ou omita limit para cálculos de soma/total
3. **PRECISÃO**: Use valores exatos do 'summary', nunca recalcule manualmente
4. **COMISSÃO**: Filtre status "pago" e exclua "Sedex" dos cálculos
5. **STATUS**: "pagos" = statuses: ["pago"]
6. **CLIENTE SEM NÚMERO**: Se usuário disser "João pagou", use get_client_orders primeiro
